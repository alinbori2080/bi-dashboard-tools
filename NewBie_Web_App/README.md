# 新锐攻略用户数据

本工具用于分析新锐攻略推送批次的用户留存和付费表现，核心报表为：

- `推送用户留存付费汇总`

工具是本地网页应用，双击 `start.cmd` 后会启动本地服务并自动打开看板。

## 访问地址

- 默认地址：`http://127.0.0.1:8766/`
- 启动方式：双击 `NewBie_Web_App/start.cmd`
- 如果服务已经启动，再次双击 `start.cmd` 会直接打开网页，不会重复启动服务。

## 日常使用

### 查看飞书结果表

1. 双击 `start.cmd`。
2. 页面默认数据源为 `飞书结果表`。
3. 点击 `刷新数据`，从飞书结果表读取当前数据。
4. 使用时间范围、对比维度、LTV 指标和留存指标查看看板。
5. 可点击 `导出 CSV` 导出当前页面筛选后的报表。

飞书结果表模式下，数据已经来自飞书，不需要也不能再次同步飞书。

### 使用本地源文件重新计算

1. 将最新源文件放入 `NewBie_Web_App/data`。
2. 页面数据源切换为 `本地文件`。
3. 点击 `刷新数据`。
4. 检查看板、趋势图和明细表。
5. 确认无误后，点击 `确认同步飞书`。

刷新只生成本地结果，不会自动写入飞书。只有点击 `确认同步飞书` 后才会写入飞书结果表。

## 源文件规则

本地刷新会读取 `data` 目录中最新的匹配文件。

- `名将杀 私域需求*.xlsx`：推送批次源数据，Sheet 名需为类似 `5.14`、`6.4` 的日期格式。
- `新登_*.csv`：GDATA 大盘新登留存数据，必须包含 `日期`、`新登账号`、`次留率`、`3留率`、`7留率`。

## 统计口径

### 推送批次

- 每个日期 Sheet 作为一个推送批次。
- 每个批次必须同时包含实验组和对照组。
- 页面中的 `当前统计批次` 按当前筛选后的批次展示。
- NewBie 的批次不是自然日连续累加，不跨周合并无关批次。

### 留存

输出字段包括：

- 次日留存率
- 3日留存率
- 7日留存率

留存按推送批次内实验组、对照组各自用户量计算。

### LTV

输出字段包括：

- `LTV7`
- `LTV15`
- `LTV30`
- `LTV7差值`
- `LTV15差值`
- `LTV30差值`

差值为实验组与对照组同口径 LTV 的差值。

### 大盘留存

大盘留存来自 `新登_*.csv`。

- 周四推送批次：匹配该批次前 6 天的新登账号，并按 `新登账号` 加权。
- 周五推送批次：匹配该批次前 1 天的新登账号。
- `新登_*.csv` 中的 `7留率` 表示对应日期新登账号在第 7 天打开游戏的比例。
- 如果对应日期段缺少日期、字段或留存值，则对应大盘字段为空，不补 0。

示例：

- `6.18` 推送用户对应 `6.12-6.17` 新登账号。
- `6.19` 推送用户对应 `6.18` 新登账号。

## 页面功能

- 数据源切换：`飞书结果表`、`本地文件`。
- 时间范围筛选：支持全部、近 7 天、近 30 天和自定义。
- 对比维度：支持周、月。
- LTV 趋势：可切换 `LTV7`、`LTV15`、`LTV30`。
- 留存趋势：可切换次留、3 留、7 留。
- 指标卡：展示用户量、LTV 差值、实验组留存、较上周、较上月、较对照组、较大盘等。
- 明细表：展示实验组和对照组明细，默认按批次降序。
- 导出 CSV：导出内容与当前页面数据源、筛选条件、排序和统计批次一致。
- 飞书链接：可在右侧飞书同步区域点击 `打开` 进入对应飞书结果表。

## 最近刷新时间

页面展示 `最后刷新`：

- 取最近一次刷新数据成功完成的时间。
- 页面打开时间不会覆盖该时间。
- 飞书同步时间不会覆盖该时间。
- 如果本次刷新失败，会保留上一次成功刷新时间。

## 飞书同步

- 飞书结果表模式：只读飞书，不允许同步。
- 本地文件模式：刷新成功后可手动同步飞书。
- 同步前会检查是否存在计算错误。
- 同步成功后页面会展示新增、更新等结果。
- 同步失败时页面展示错误原因。

## 目录说明

- `data/`：本地源数据目录。
- `output/`：本地刷新后生成 `推送用户留存付费汇总.csv`。
- `config/`：飞书配置和同步状态。
- `static/`：网页前端文件。
- `runtime_state.json`：最近一次页面运行状态。
- `start.cmd`：本地启动脚本。

## 常见问题

### 大盘 7 日留存为空

先确认对应推送批次映射的新登日期段是否完整。如果缺少日期、字段或 `7留率`，对应大盘 7 留会保持为空。

### 双击 start.cmd 没反应

确认电脑已安装 Python，并且命令行中可执行：

```text
python --version
```

### 页面显示飞书读取失败

检查飞书配置、网络、应用权限和结果表字段是否可用。
