# BI 看板工具部署说明

本文档用于把仓库中的 BI 看板工具部署到服务器，方便其他人打开网页即可使用最新版本。

当前已完成标准化部署流程的工具：

- `30Day_Web_App`：30 日留存数据看板
- `NewBie_Web_App`：新锐攻略用户数据看板

## 一、部署目标

部署后的目标效果：

- 用户通过浏览器访问服务器地址即可打开看板。
- 前端页面来自 `30Day_Web_App/frontend/dist` 构建产物。
- 飞书密钥和表格配置只放服务器环境变量，不提交到 GitHub。
- 每次更新代码后，服务器重新构建前端并重启服务即可使用最新版。

## 二、服务器环境要求

服务器需要具备：

- Git
- Python 3.12 或更高版本
- Node.js 22 或更高版本
- npm

检查命令：

```bash
git --version
python --version
node --version
npm --version
```

## 三、首次部署流程

### 1. 拉取代码

```bash
git clone https://github.com/alinbori2080/bi-dashboard-tools.git
cd bi-dashboard-tools
```

### 2. 配置飞书环境变量

在服务器环境中配置以下变量：

```bash
FEISHU_APP_ID=你的飞书应用ID
FEISHU_APP_SECRET=你的飞书应用密钥
FEISHU_APP_TOKEN=你的飞书多维表格AppToken
FEISHU_DAILY_TABLE_ID=30日留存结果表ID
FEISHU_APP_URL=飞书表格或文档链接
```

注意：

- 不要把真实密钥写入仓库。
- 不要把真实密钥写入前端 `.env`。
- 不要提交 `config/*.json`。
- 如果服务器使用系统服务管理，需要把这些环境变量配置到服务进程能读取的位置。

如果必须使用 `30Day_Web_App/config/feishu_sync_config.json`，当前 30 日留存工具只保留 `daily` 一张结果表，标准结构如下：

```json
{
  "app_id": "你的飞书应用ID",
  "app_secret": "你的飞书应用密钥",
  "app_token": "你的飞书多维表格AppToken",
  "app_url": "飞书表格或文档链接",
  "base_name": "名将杀30日留存",
  "table_names": {
    "daily": "每日结果"
  },
  "table_ids": {
    "daily": "30日留存结果表ID"
  }
}
```

旧配置中的 `"summary": "汇总结果"` 和 `summary` 表 ID 已废弃，不需要保留。

### 3. 构建 30 日留存前端

```bash
cd 30Day_Web_App/frontend
npm ci
npm run check
cd ../..
```

说明：

- `npm ci`：按 `package-lock.json` 安装依赖。
- `npm run check`：执行类型检查、前端单测、前端构建和服务冒烟测试。
- 构建成功后会生成 `30Day_Web_App/frontend/dist`。

如果部署新锐攻略用户数据看板，构建命令为：

```bash
cd NewBie_Web_App/frontend
npm ci
npm run check
cd ../..
```

构建成功后会生成 `NewBie_Web_App/frontend/dist`。

### 4. 启动后端服务

```bash
cd 30Day_Web_App
python server.py 8767
```

启动后访问：

```text
http://服务器IP:8767/
```

如果部署在内网服务器，也可以让运维同事配置反向代理，把外部域名转发到 `127.0.0.1:8767`。

新锐攻略用户数据看板默认端口为 `8766`：

```bash
cd NewBie_Web_App
python server.py 8766
```

## 四、日常更新流程

服务器更新到 GitHub 最新版本：

```bash
cd bi-dashboard-tools
git pull
cd 30Day_Web_App/frontend
npm ci
npm run check
cd ..
python server.py 8767
```

如果服务已经由系统服务、Supervisor、PM2 或 Docker 管理，则更新后重启对应服务。

新锐攻略用户数据看板更新时，把路径替换为 `NewBie_Web_App/frontend`，并重启 `NewBie_Web_App/server.py`。

## 五、验证方法

### 1. GitHub 自动检查

每次推送到 `main` 后，GitHub Actions 会自动运行：

- 前端类型检查
- 前端单元测试
- 前端构建
- 服务冒烟测试
- Python 单测
- 前端入口结构回归测试

查看地址：

```text
https://github.com/alinbori2080/bi-dashboard-tools/actions
```

绿色对勾表示通过。

### 2. 服务器本地检查

在服务器运行：

```bash
cd bi-dashboard-tools
npm run check --prefix 30Day_Web_App/frontend
python -m unittest discover -s 30Day_Web_App/tests -p "test_*.py"
node 30Day_Web_App/tests/frontend_previous_month.test.js
```

全部通过后，再重启服务。

### 3. 页面检查

打开看板后确认：

- 页面能正常加载。
- 页面没有显示“静态回退提示页”。
- 数据源可切换为 `飞书结果表` 和 `本地结果文件`。
- 30 日留存明细、趋势图、导出全部按钮正常。
- 右侧飞书同步状态正常显示。

## 七、回滚流程

如果新版本上线后异常：

```bash
cd bi-dashboard-tools
git log --oneline
git checkout 上一个稳定提交ID
cd 30Day_Web_App/frontend
npm ci
npm run check
cd ..
python server.py 8767
```

确认旧版本恢复后，再决定是否回到 `main` 并修复问题。

恢复到最新版本：

```bash
cd bi-dashboard-tools
git checkout main
git pull
```

## 八、常见问题

### 页面显示静态回退提示页

原因：`30Day_Web_App/frontend/dist/index.html` 不存在。

处理：

```bash
cd 30Day_Web_App/frontend
npm ci
npm run build
```

### 飞书读取失败

优先检查：

- 服务器是否能访问飞书接口。
- 环境变量是否配置完整。
- 飞书应用权限是否包含目标多维表格。
- `FEISHU_APP_TOKEN` 和 `FEISHU_DAILY_TABLE_ID` 是否正确。

### GitHub Actions 失败

进入失败任务查看日志：

```text
https://github.com/alinbori2080/bi-dashboard-tools/actions
```

常见原因：

- 前端类型错误。
- 测试失败。
- 构建失败。
- 服务冒烟测试启动失败。

先修复失败项，再重新推送。

## 九、安全要求

- 真实飞书密钥只能放服务器环境变量。
- 不要提交 `config/*.json`。
- 不要提交 `runtime_state.json`。
- 不要提交 `data/` 下的真实源数据。
- 不要提交 `output/` 下的生成结果。
- 不要把密钥写进前端代码或前端环境变量。
